隨心現場錄音唸佛助手 (Buddha Chanting Assistant)
一個輕量、純前端的網頁版唸佛/持咒隨身助手。專為需要靈活記錄、循環播放自己或法師現場法音的修持者設計。本專案特別針對移動端（尤其是 iOS Safari）的音訊播放限制進行了優化，確保在鎖屏或連續播放時能流暢無縫地循環。

🚀 項目已啟用 GitHub Pages 部署，即開即用。

🌟 核心功能
🎙️ 現場隨心錄音：只需按下錄音鍵，即可現場錄製一句專屬的佛號、聖號或咒語（如：阿彌陀佛、觀音聖號、六字大明咒等）。

🔄 雙重播經模式：

設定次數：內建 10 次、21 次、49 次、108 次及 1000 次等傳統功德圓滿計數。

無限循環：無限制持續播放，適合靜坐、定時修持或環境佛號常持。

⚡ 毫秒級無縫重播：採用 HTML5 Audio 事件驅動優化，避開移動端常見的音訊延遲與卡頓，實現流暢的連播體驗。

📊 功德統計紀錄：利用瀏覽器 localStorage 本地儲存每次修持的日期、次數及是否圓滿，數據不外傳，完美保護個人隱私。

📱 完美相容 iOS/Android：特別處理過原生播放器相容性，強制音量輸出，並在功德圓滿時提供設備震動反饋（需裝置支援）。

🛠️ 技術亮點
此專案雖然是一個單純的 index.html 檔案，但在移動端音訊相容性上做了精準優化：

iOS 標準 Blob URL 模式：優先強制採用 audio/mp4 編碼錄製，自動釋放舊物件記憶體（revoking URL），徹底解決 iPhone 網頁錄音無法播放的通病。

繞過 Web Audio API Bug：棄用容易在 iOS 鎖屏或背景斷音的 Web Audio 節點控制，改用純 HTML5 原生事件監聽（ended 事件驅動），達成超低延遲的重播。

零後端依賴：無任何第三方 API 或伺服器交互，100% 本地安全運行。

📖 使用方法
打開網頁：進入部署好的網頁。

第一步（錄音）：點擊 「🔴 開始錄音 (講一句)」，對著麥克風念誦一聲佛號，完成後再次點擊停止。

第二步（設定）：選擇「設定次數」或「無限循環」，並挑選您的功德目標次數。

第三步（開始）：點擊 「開始連播」，系統將自動開始循環播放並即時累加計數。

查看紀錄：頁面下方會即時顯示歷史修持日誌，您可以隨時點擊「清除紀錄」重設。

💡 iOS 使用小提示：因 Apple 安全機制限制，首次開始連播時必須由用戶主動點擊「開始連播」按鈕以觸發音訊。請確保手機未處於「靜音模式」，以免音訊被系統阻斷。

⚙️ 本地開發與部署
快速部署至 GitHub Pages
因為本專案完全由靜態的 index.html 組成，你可以非常輕鬆地將它上線：

將程式碼推送到 GitHub 倉庫。

進入倉庫的 Settings -> Pages。

在 Build and deployment 下方的 Build 來源選擇 Deploy from a branch。

選擇 main 分支與 / (root) 目錄，點擊 Save。

幾分鐘後，你就能透過 https://<你的用戶名>.github.io/Buddha/ 訪問專案。

本地運行
只需複製本專案，並直接用瀏覽器打開 index.html 即可運行。
若需測試錄音功能，部分瀏覽器基於安全策略要求必須在 localhost 或 https:// 安全協定下才允許調用麥克風權限。
